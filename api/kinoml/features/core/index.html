    kinoml.features.core¶

    Featurizers can transform a kinoml.core.system.System object and produce new representations of the molecular entities and their associated measurements.

    All Featurizer objects inherit from BaseFeaturizer and reimplement ._featurize and ._supports, if needed.

    Module Contents¶

    Abstract Featurizer class

    _SUPPORTED_TYPES¶
    featurize(self, system, inplace: bool = True) → object¶

    Given an system (compatible with _SUPPORTED_TYPES), apply the featurization scheme implemented in this class.

    Parameters

    • system (System) – This is System object that will be transformed

    • inplace (bool, optional) – Whether to modify the System directly or operate on a copy.

    Returns

    system – The same system that was passed in, unless inplace is False. The returned System will have an extra entry in the .featurizations dictionary, containing the featurized object (either a new System or an array-like object) undera key named after .name.

    Return type

    System

    Check if these systems are supported by this featurizer.

    Do NOT reimplement in subclass. Check ._supports() instead.

    _supports(self, system: System) → bool¶

    This is the method you should reimplement in your subclass.
